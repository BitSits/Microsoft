    Exercise 3: Launch Another Application from the Game

    In this exercise, we add a new feature to the game, the ability to launch other applications.

    As we already mentioned, Windows Phone allows only one application to run at a time. At the same time, you can’t just launch other applications, nor can you access phone-specific information or applications like the contact list. You need special means to launch other applications from your running game. The Choosers and Launchers enable you to use phone-specific applications to access specific data or use the phone capabilities such as the camera. To learn more about choosers and launchers, see the Launchers and Choosers Hand-On-Lab from this Training Kit.

    Note that your application will not run when another application runs. Moreover, the phone should know that we did not exit our application, but just switched to another one and may still come back. Tombstone is the state that we handled in the last exercise, and it is a good thing that we did so, because when you are launching specific choosers and launchers your application will get tombstoned.

    Task 1 –Run another application from the game

    As we already mentioned, we use Choosers and Launchers to launch another application from our game. The difference between Choosers and Launchers is that Choosers provide results back to the game, while Launchers do not. The development platform provides a rich collection of Choosers and Launchers that launch standard Windows Phone tasks out-of-the box. In this exercise, we will use a Chooser of type PhoneNumberChooserTask and a Launcher of type SmsComposeTask.

    PhoneNumberChooserTask launches the standard Phone Book application and returns the selected number back to the game. SmsComposeTask sends SMS to the given number. In this task, you will integrate two those capabilities into our game.

    1. Open file CatapultGame.cs.
    2. Add the following statement to the "Using Statements" region:
      1. using Microsoft.Phone.Tasks;

      Note:
      This statement allows us use Choosers and Launchers.

    3. Add the following field to the "Fields" region:

      (Code Snippet – 2D Game Development with XNA Part 2 – CatapultGame Fields)

      1. PhoneNumberChooserTask phoneNumberChooserTask;SmsComposeTask smsComposeTask;

      Note:
      In those fields we will store our Choose and Launcher objects.

    4. Locate the CatapultGame constructor and replace it with the following code:

      (Code Snippet – 2D Game Development with XNA Part 2 – CatapultGame Constructor)

      1. public CatapultGame(){ graphics = new GraphicsDeviceManager(this); Content.RootDirectory = "Content"; // Frame rate is 30 fps by default for Windows Phone. TargetElapsedTime = TimeSpan.FromTicks(333333); //Create a new instance of the Screen Manager screenManager = new ScreenManager(this); Components.Add(screenManager); //Switch to full screen for best game experience graphics.IsFullScreen = true; //Add two new screens screenManager.AddScreen(new BackgroundScreen(), null); screenManager.AddScreen(new MainMenuScreen(), null); //Create Chooser and Launcher InitializeChooserAndLauncher(); AudioManager.Initialize(this); InitializePhoneServices();}

      Note:
      Notice the highlighted statement. It initializes the Chooser and Launcher objects.

    5. Add the following code to the "Initialization Methods" region:

      (Code Snippet – 2D Game Development with XNA Part 2 – CatapultGame InitializeChooserAndLauncher)

      1. private void InitializeChooserAndLauncher(){ phoneNumberChooserTask = new PhoneNumberChooserTask(); smsComposeTask = new SmsComposeTask(); phoneNumberChooserTask.Completed += new EventHandler<PhoneNumberResult>(PhoneNumberChooserTaskCompleted);}

      Note:
      This code initializes our Chooser and Launcher.

    6. Add the following code to the "Event Handlers" region:

      (Code Snippet – 2D Game Development with XNA Part 2 – CatapultGame Event Handlers)

      1. /// <summary>/// Occurs when the Phone Number Chooser task completes/// </summary>/// <param name="sender"></param>/// <param name="args"></param>void PhoneNumberChooserTaskCompleted(object sender, PhoneNumberResult args){ // If user provided the requested info if (args.TaskResult == TaskResult.OK) { // Create, initialize and show SMS composer launcher smsComposeTask.To = args.PhoneNumber; smsComposeTask.Body = "Hello! Just discovered very good game called Catapult Wars. Try it by yourself and see!"; smsComposeTask.Show(); }}

      Note:
      This event handler completes the "Phone Number Chooser" task by launching the SMS Composer with the chosen number.

    7. Add the following code to the CatapultGame class:

      (Code Snippet – 2D Game Development with XNA Part 2 – CatapultGame ExecuteTask)

      1. #region Public Interfacepublic void ExecuteTask(){ phoneNumberChooserTask.Show();}#endregion

      Note:
      This code allows invoke our Chooser and Launcher from the menu, as we will see below.

    8. Open file MainMenuScreen.cs located in “Screens” folder.
    9. Locate the constructor and replace it with the following code:

      (Code Snippet – 2D Game Development with XNA Part 2 – MainMenuScreen Constructor)

      1. public MainMenuScreen() : base(String.Empty){ IsPopup = true; // Create our menu entries. MenuEntry startGameMenuEntry = new MenuEntry("Play"); MenuEntry selectBackgroundMusic = new MenuEntry("Select Background Music"); MenuEntry shareMenuEntry = new MenuEntry("Tell a Friend"); MenuEntry exitMenuEntry = new MenuEntry("Exit"); // Hook up menu event handlers. startGameMenuEntry.Selected += StartGameMenuEntrySelected; selectBackgroundMusic.Selected += SelectBackgroundMusicMenuEntrySelected; shareMenuEntry.Selected += ShareMenuEntrySelected; exitMenuEntry.Selected += OnCancel; // Add entries to the menu. MenuEntries.Add(startGameMenuEntry); MenuEntries.Add(selectBackgroundMusic); MenuEntries.Add(shareMenuEntry); MenuEntries.Add(exitMenuEntry);}

      Note:
      Notice the highlighted statements. Those statements introduce a new menu item that will launch the Phone Book and send SMS to the chosen number.

    10. Locate “Event Handlers for Menu Items” and add there the following code:

      (Code Snippet – 2D Game Development with XNA Part 2 – MainMenuScreen ShareMenuEntrySelected)

      1. /// <summary>/// Handler "Share" menu item selection/// </summary>/// <param name="sender"></param>/// <param name="e"></param>void ShareMenuEntrySelected(object sender, EventArgs e){ //Execute a task defined in the game class ((CatapultGame)(ScreenManager.Game)).ExecuteTask();}

      Note:
      This is the event handler for the new menu item. This handler executes the task defined in the game application, as described above. This handler launches the Phone Book application, gets back the chosen number and sends an SMS to this number.

    11. This concludes the task and the exercise.
    12. Now our game player can share scores with a friend via SMS.
